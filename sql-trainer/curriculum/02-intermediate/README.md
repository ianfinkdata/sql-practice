# Tier 2 · Intermediate

Welcome back. In Tier 1 you learned to pull data from a single table — selecting columns, filtering rows, and sorting results. That's a solid foundation, but real databases don't keep everything in one neat table, and real questions rarely have one-row answers.

Tier 2 is where SQL gets genuinely powerful. Here you'll learn to **combine tables, summarize data, and handle the messy parts** — the missing values, the edge cases, and the conditional logic that show up in every real-world dataset. By the end, you'll be able to answer the kind of layered, multi-table questions that businesses actually ask.

Work through the modules in order; each one builds on the last.

## Modules

1. **[Aggregate Functions](./01-aggregate-functions.md)** — COUNT, SUM, AVG, MIN, and MAX, and how they collapse many rows into a single summary number.
2. **[Group By](./02-group-by.md)** — Summarize per category, one result per group, and the rule every beginner trips on.
3. **[HAVING](./03-having.md)** — Filter whole groups after summarizing, and how it differs from WHERE.
4. **[Joins: Inner & Left](./04-joins-inner-left.md)** — Relate tables on shared keys; INNER vs LEFT, ON vs WHERE, and aliases.
5. **[Joins: Advanced](./05-joins-advanced.md)** — RIGHT, FULL OUTER, CROSS, and SELF joins, plus the indispensable anti-join.
6. **[Set Operations](./06-set-operations.md)** — Combine, overlap, and subtract result sets with UNION, INTERSECT, and EXCEPT.
7. **[Subqueries](./07-subqueries.md)** — Queries inside queries: scalar, nested, and correlated forms, and where they go.
8. **[CASE & Conditional Logic](./08-case-conditional.md)** — If-this-then-that in SQL, including powerful conditional aggregation.
9. **[Working with NULLs](./09-nulls.md)** — The "unknown" value, three-valued logic, and the tools to tame it.
